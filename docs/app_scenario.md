# 앱 사용 시나리오 & 개발 계획서

> 발달장애인 돌봄 에이전트 AI — SKKU URP 26-1

## 1. 서비스 개요

### 핵심 목표
발달장애인(경도·중등도 지적장애 / ASD)의 **일관된 스케줄 및 루틴 관리**를 AI가 지원하여
독립적인 일상생활 능력을 향상시킨다.

### 타깃 사용자 (서비스 대상)

| 구분 | 대상 | 제외 |
|------|------|------|
| 당사자 | 경도·중등도 지적장애인 / ASD | 고도·최고도 (독립적 앱 조작 불가) |
| 보호자 | 부모, 가족, 생활 지원사 | — |
| 기관 | 복지관, 치료센터, 병원 담당자 | — |

> 근거: 경도(9~12세 수준, 한국 지적장애인 80%), 중등도(4~8세 수준, 12%)만 앱 자체 조작 가능

## 2. 서비스 설계 원칙 (문헌 기반)

| 원칙 | 내용 | 근거 |
|------|------|------|
| Easy Read + Multimodal | 짧은 문장 + 이미지 + 음성 동시 제공 | Bruland et al., 2025 |
| 인지 과부화 방지 | 한 번에 한 가지 정보만, 단순 문장 | Van der Molen et al., 2010 |
| 긍정적 행동 지원(PBS) | 부정적 피드백 금지, 선택지로 자율성 보장 | 긍정적 행동중재 연구 |
| 자기 점검 | 스스로 달성 여부 확인 → 동기부여 | 자기모델링/자기점검표 연구 |
| 루틴 변경 사전 알림 | 변경 발생 시 충분히 미리 고지 | COM-B 모델 |
| 상황형 애니메이션 이모티콘 | 구체적 상황 + 움직이는 이미지 조합 | 이영규·김현석, 2023 |

## 3. 플랫폼 초기 설정

```
보호자 앱에서 입력 (최초 1회)
│
├── 1. 사용자 장애 종류/정도 설정
│       지적장애 경도 / 지적장애 중등도 / ASD
│
├── 2. 개인정보 및 특이사항 입력
│       - 이름, 나이
│       - 특이사항: 특정 활동 선호/비선호, 감각 민감성
│       - 주 1회 특정 장소 방문 여부
│       - 매일 반드시 해야 하는 활동
│       - 알려진 전조증상 (보호자가 직접 입력)
│
└── 3. AI 자동 설정
        - 디폴트 스케줄표 생성 (장애 종류/정도 기반)
        - 개인 특이사항 반영하여 수정 가능
        - 피드백 전달 방식 자동 결정:
            ASD → AAC 버튼 우선
            경도 지적장애 → 음성
            중등도 지적장애 → 텍스트+이미지
```

## 4. 5개 Tool 구성 및 역할

```
┌─────────────────────────────────────────────────────┐
│                  AI Care Agent                       │
│                 (LLaMA 3.1 기반)                     │
└──────┬──────┬──────┬──────┬──────────────────────────┘
       │      │      │      │
  ┌────▼──┐ ┌─▼────┐ ┌▼────┐ ┌▼─────────┐ ┌──────────┐
  │Tool 1 │ │Tool 2│ │Tool3│ │  Tool 4  │ │  Tool 5  │
  │사용자 │ │스케줄│ │단계별│ │ 사용자   │ │ 메시지   │
  │개별화 │ │달성  │ │피드백│ │ 반응     │ │ 발신     │
  │       │ │확인  │ │      │ │ Detect   │ │          │
  └───────┘ └──────┘ └──────┘ └──────────┘ └──────────┘
```

| Tool | 역할 | 호출 시점 |
|------|------|-----------|
| Tool 1 사용자 개별화 | 사용자 정보 → 스케줄표 구성 + 피드백 방식 결정 | 앱 초기 설정 시 |
| Tool 2 스케줄 달성 확인 | 달성량 체크 + 보호자 메시지 발신 Tool 호출 | 스케줄 시간 30분 후 |
| Tool 3 단계별 피드백 | Detect Tool 결과에 따라 1/2/3단계 피드백 실행 | Detect Tool 이후 |
| Tool 4 사용자 반응 Detect | 음성 데시벨/GPS/텍스트로 상태 감지 | 스케줄 공지 후 주기적 |
| Tool 5 메시지 발신 | 일과 달성률 / 긴급 알림 발송 | 매일 밤 + RED 단계 |

## 5. 사용자 반응 Detect 방식

| 감지 수단 | 내용 | 임계값 |
|-----------|------|--------|
| 음성 (마이크) | 데시벨로 흥분도 측정 | 50dB 이하: Green / 50~85dB: Amber / 85dB↑: Red |
| GPS | 이동 스케줄 탐지 (장소 이동 여부) | 30분 후 미이동 = 스케줄 미달성 신호 |
| 텍스트 키워드 | 거부 표현 감지 | "싫어", "안해", "못해" 등 |
| 영상 탐지 | 경기도 X SKT 사례 참고 | ⚠️ 프라이버시 이슈로 추후 검토 |

## 6. 단계별 피드백 방식 (PBS 기반)

```
Green ──────────────────────────────────────────
│ 상태: 문제 행동 없음, 정상 수행
│ 대응: 정상 스케줄 안내, 완료 시 긍정적 피드백
│
Amber (1단계) ────────────────────────────────
│ 감지: 데시벨 50~85dB / 텍스트 거부 / "싫어요" 버튼
│
│ 대응 ①  선택지 2개 제시 (둘 다 수용 가능한 것)
│          "지금 할까요, 3분 뒤에 할까요?"
│          "세면대에서 할까요, 컵으로 할까요?"
│
│ 대응 ②  과제 쪼개기
│          "치약만 올려볼까요? (1단계만)"
│
│ 대응 ③  FCT 대체 의사소통 버튼 (앱 UI)
│          [쉬고 싶어요] [도와주세요] [싫어요 대신 ○○할래요]
│          → 문제행동 대신 대체 행동으로 불만 해소 (FCT)
│
Red (2단계) ──────────────────────────────────
│ 감지: 데시벨 85dB↑ / 소리 지르거나 과격한 반응
│
│ 대응 ①  낮고 일정한 모노톤으로 메시지 전달
│          "괜찮아요. 잠깐 쉬어요."
│
│ 대응 ②  행동 전환 유도
│          "좋아하는 것을 잠깐 해볼까요?"
│
│ 대응 ③  보호자/기관에 즉시 연락 (메시지 발신 Tool)
│
진정 후 (3단계) ──────────────────────────────
│ 감지: 흥분 상태 해소 (데시벨 정상화 or 보호자 확인)
│
│ 즉시: 안전 점검
│       "다친 곳은 없나요?"
│
│ 1시간 후: 대화 시도
│       "아까 어떤 기분이었는지 이야기해볼까요?"
│       쉬운 질문부터 시작 → 괜찮으면 심화
│       "오늘 힘든 게 있었나요?"
│       "무엇 때문에 힘들었나요?"
│       "다음엔 어떻게 하면 좋을까요?"
│       → 사용자 성향 학습 (다음번 전조증상 예측에 활용)
```

## 7. 하루 흐름 시나리오

### 시나리오 A — 정상 흐름

```
전날 밤 (예: 21:00)
  AI → 당사자 앱: "내일 할 일을 알려드릴게요!"
                   [스케줄 카드 슬라이드]
                   🍳 09:00 아침 식사
                   🦷 10:00 양치하기
                   🚶 13:00 산책
  AI → 보호자 앱: "내일 ○○이 스케줄을 확인해보세요" (다음날 스케줄)

AM 09:00 (스케줄 시간)
  [사용자 개별화 Tool 실행] → 사용자 프로필 확인, 피드백 방식 로드
  AI → 당사자: "아침 식사 하실 시간이에요 🍳"
               [이미지: 밥상 애니메이션]
  당사자 → AI: (버튼) ✅ 할게요

AM 09:30 (30분 후 달성 확인)
  [스케줄 달성 확인 Tool 실행]
  AI → 당사자: "아침 식사는 뭐 드셨어요?"

  ✅ Yes (달성):
    AI: "잘했어요! 😊"
    [메시지 발신 Tool] → 보호자: "○○이가 아침 식사를 마쳤어요 ✅"

  ❌ No (미달성):
    AI: "뭐 하셨나요?" (자연스럽게 대화하며 개인화 정보 취득)
    → 내일 스케줄 수정에 반영
    [메시지 발신 Tool] → 보호자: "○○이가 아침 식사를 하지 않았어요"
```

### 시나리오 B — 문제 행동 발생

```
PM 13:00 (산책 스케줄)
  [사용자 개별화 Tool] → 사용자 프로필 로드
  AI → 당사자: "이제 산책 시간이에요 🚶"

  당사자 → AI: "싫어!"

  [사용자 반응 Detect Tool]
  → 텍스트 거부 키워드 감지 → Amber (1단계) 판정

  [단계별 피드백 Tool — 1단계 실행]
  AI → 당사자: "지금 할까요, 10분 뒤에 할까요?"
               [버튼] 🕐 지금 할게요  |  ⏳ 10분 뒤에 할게요
               [버튼] 😴 쉬고 싶어요  |  🙋 도와주세요

  → 당사자: 소리 지르거나 과격한 반응

  [사용자 반응 Detect Tool]
  → 데시벨 85dB↑ 감지 → Red (2단계) 판정

  [단계별 피드백 Tool — 2단계 실행]
  AI → 당사자: (낮고 일정한 톤) "괜찮아요. 잠깐 쉬어요."
               "좋아하는 것을 잠깐 해볼까요?"
  [메시지 발신 Tool] → 보호자/기관: "⚠️ ○○이가 흥분 상태입니다. 확인 바랍니다."

  (흥분 진정 후)
  [단계별 피드백 Tool — 3단계 실행]
  AI → 당사자: "다친 곳은 없나요?"
  (1시간 후)
  AI → 당사자: "아까 어떤 기분이었는지 이야기해볼까요?"
```

### 시나리오 C — 스케줄 미달성 (GPS 기반)

```
PM 13:00 (산책 스케줄)
  [사용자 개별화 Tool] → GPS 활성화, 이동 스케줄 확인
  AI → 당사자: "이제 산책 시간이에요 🚶"

  PM 13:30 (30분 후)
  [사용자 반응 Detect Tool] → GPS 미이동 감지

  [단계별 피드백 Tool — 1단계 실행]
  AI → 당사자: "산책 안 가셨나요? 지금 나가볼까요?"
               (쉬운 문장 / 상냥한 말투)

  → 당사자: 소리 지르거나 과격한 반응

  [사용자 반응 Detect Tool] → 데시벨 85dB↑ → Red (2단계)
  [단계별 피드백 Tool — 2단계 실행] + 보호자 알림
```

## 8. 보호자 앱 기능

### 초기 설정
- 당사자 프로필 등록 (장애 종류/정도, 특이사항)
- 스케줄표 구성 (시간 + 활동 + 이미지 위젯)
- 알려진 전조증상 입력 (AI가 Amber 감지에 활용)

### 일상 사용
```
[실시간 모니터링]
  - 현재 스케줄 진행 상황 (Green / Amber / Red 상태)
  - 오늘 달성률 (%)
  - 마지막 확인된 상태

[알림 수신]
  - Amber 진입: "○○이가 산책을 거부하고 있어요"
  - Red 발생: 즉시 긴급 알림 + 상황 내용
  - 스케줄 미달성: "○○이가 산책을 하지 않았어요"

[일일 리포트 (매일 밤)]
  - 오늘 달성률 + 미달성 항목
  - 문제 행동 발생 횟수/시간대
  - 내일 스케줄 미리보기
```

### 스케줄 변경 시
```
보호자가 스케줄 변경 입력
  → AI가 당사자에게 사전 알림 자동 발송
  → D-3일, D-1일, 당일 아침 반복 고지
  (근거: 루틴 변경 시 충분한 경고가 있다면 수용 가능 — COM-B 연구)
```

- [ ] 전조증상 DB 필드 추가 (보호자 입력 → AI 활용)

- [ ] 보호자 — 실시간 모니터링 화면 (Green/Amber/Red)
- [ ] 보호자 — 일일 리포트 화면
- [ ] 보호자 — 스케줄 설정 화면

### Phase 2 — 고도화 (P1)
- [ ] 전조증상 패턴 학습 (3단계 대화 후 AI 자동 학습)

- [ ] 자기 점검 체크리스트 (당사자 스스로 달성 체크)
- [ ] 보상 시스템 (포인트/스티커/뱃지)

- [ ] 보호자/전문가 대상 설문조사 실시
- [ ] 실제 사용자 파일럿 테스트

## 10. 미결 논의 사항

| 항목 | 현황 | 결정 방향 |
|------|------|-----------|
| 전조증상 사전 예방 | 보호자가 수동 입력 or AI가 3단계 대화로 학습 | 두 방식 병행 검토 |
| 보호자 정보 공유 범위 | 루틴에이드 연구에서 당사자 거부감 확인 | 당사자가 공유 범위 선택 가능하게 |
| 영상 탐지 도입 여부 | SKT 사례 참고 가능하나 프라이버시 이슈 | Phase 3 이후 검토 |
| 동기부여 방식 | 보상 시스템 구체화 필요 | 캐릭터 성장 or 뱃지 갤러리 |
| 기관 역할 범위 | 아직 미정 | 보호자/전문가 설문 후 결정 |
| 디폴트 스케줄표 | 이상적인 발달장애인 스케줄표 필요 | 전문가 자문 or 문헌 기반 구성 |
